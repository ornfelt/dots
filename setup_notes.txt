# mysql setup

sudo pacman -S mariadb

Setup the data directory:
sudo mariadb-install-db --user=mysql --basedir=/usr --datadir=/var/lib/mysql

Start mariadb.service
systemctl start mariadb.service
systemctl enable mariadb.service

Add user
In the below example, the user monty with some~pass~ as password is being
created, then granted full permissions to the database mydb:
sudo mysql -u root -p
MariaDB> CREATE USER 'monty'@'localhost' IDENTIFIED BY 'some_pass';
MariaDB> GRANT ALL PRIVILEGES ON mydb.* TO 'monty'@'localhost';
MariaDB> FLUSH PRIVILEGES;
MariaDB> quit

Do this instead (to allow mysql root without sudo):
sudo mysql -u root -p
ALTER USER 'root'@'localhost' IDENTIFIED BY '...';
FLUSH PRIVILEGES;

then:
mysql -u root -p
...

Debug:
Check authentication plugin
Ensure the root user is using the mysql_native_password or
caching_sha2_password plugin (instead of unix_socket):
SELECT user, host, plugin FROM mysql.user;

If you see unix_socket for root, update it:
ALTER USER 'root'@'localhost' IDENTIFIED WITH mysql_native_password BY '...';
FLUSH PRIVILEGES;

To suppress or prevent the mysql/mariadb deprecation warning:
Option 1: Use mariadb directly:
mariadb -u root -p
Or:
alias mysql='mariadb'
Add to ~/.bashrc or ~/.zshrc...
